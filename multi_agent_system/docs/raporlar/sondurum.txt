BÖLÜM C: ÇEKIRDEK ALTYAPI BİLEŞENLERİ
────────────────────────────────────────────────────────────────────────────────

C.1: Orchestrator — core/orchestrator.py
  Sistemin ana beyin merkezi. Aşağıdaki işlemleri yönetir:

  • ReAct Döngüsü: Maksimum 10 iterasyon (MAX_ITERATIONS=10) içinde görevleri
    sırayla çalıştırır.
  • DAG Bağımlılık Yönetimi: Her Task'ın 'dependencies' listesine göre hazır
    olmayan görevler askıda (PENDING) bekler, asyncio.gather ile paralel koşar.
  • Retry Zırhı: Her görev en fazla 3 kez (MAX_RETRIES=3) yeniden denenir.
    Hatanın Traceback'i bir sonraki denemede ajanın Short-Term Memory'sine enjekte
    edilir (Self-Correction mantığı).
  • _pre_install_project_deps: Coder'ın kendi oluşturduğu proje modüllerini
    (database.py, models.py vb.) pip install etmeye kalkmasını engelleyen STDLIB
    Filtresi. Projedeki tüm .py dosyaları Regex ile taranarak yerel modüller
    "bilinen kütüphane" listesine alınır.
  • Lint Entegrasyonu: Pylint ve Flake8 çıktısı 10 üzerinden normalize edilip
    Git commit mesajına eklenir (örn: "Pylint: 7.5/10 | Flake8: 0 hata")
  • Git Otomasyonu: Başarılı her görevden sonra sistem otomatik commit yapar.


C.2: Message Bus — core/message_bus.py
  Ajanların birbirini doğrudan import etmeden iletişim kurduğu Event-Driven
  (Olay Güdümlü) bellek-içi (in-memory) kuyruk sistemi.

  • Altyapı: asyncio.PriorityQueue (CRITICAL > HIGH > NORMAL > LOW)
  • Pub/Sub Pattern: Ajanlar bus.subscribe("event_name", handler) ile olaylara
    abone olur, bus.publish("event_name", data) ile event fırlatır.
  • Lose Coupling: Coder, TesterAgent'ın varlığından haberdar değildir. Sadece
    "code_written" eventi yayımlar; Tester bunu kuyruğun arkasından yakalar.
  • Faydası: Spagetti Kod / Sıkı Bağlılık (Tight Coupling) riskini ortadan
    kaldırarak sisteme yeni ajan eklemeyi kolaylaştırır.


C.3: Dosya İzolasyonu (Sandbox) — tools/file_manager.py
  Her proje workspace/projects/{slug}/ içinde izole edilmiştir.

  Güvenlik Katmanı:
    Her okuma/yazma/listele işleminde şu kontrol yapılır:
      if not resolved_path.startswith(project_root):
          raise PermissionError("Proje dışına erişim yasak!")

  Sonuç: Ajan kodu hiçbir zaman kullanıcının Desktop, Documents veya sistem

C.4: LLM İstemcisi — core/llm_client.py
  Üç farklı LLM sağlayıcısını (OpenRouter, Groq, Cerebras) tek tip bir arayüzle
  yönetir.

  • Model Bazlı Timeout: Her model için ayrı timeout ayarı vardır:

BÖLÜM I: GELECEK GELİŞTİRME YOLU HARİTASI
────────────────────────────────────────────────────────────────────────────────

Öncelik 1 — Unified Diff Format (En Acil)
  Ajanların tüm dosyayı yeniden yazmak yerine sadece değişen satırları (+/-) 
  "git diff" formatında göndermesi. Uzun dosyalarda token maliyeti %90 düşer.
  Uygulama: Coder output format'ı [DIFF:dosya.py] ... [/DIFF] bloğuna dönüştürülür.

Öncelik 2 — Debugger Agent
  Coder 3 denemede hatayı çözemezse araya giren özel bir ajan. Görevi koda
  dokunmadan geçici print() ve logging satırları ekleyip runtime değişken
  değerlerini loglamak. Bu, karmaşık runtime bug'larını görünür kılar.

Öncelik 3 — Vektör Hafıza (RAG)
  Mevcut JSON tabanlı hafızayı ChromaDB veya Pinecone ile değiştirmek.
  Coder "Bu projedeki veritabanı bağlantısı nasıl?" sorusunu semantik olarak
  sorgulayıp binlerce satır yerine ilgili 50 satırı context'e alabilir.

Öncelik 4 — Browser/GUI Testi (Playwright)
  Frontend (HTML/CSS/JS) projeleri için Playwright ile tarayıcı açıp ekran görüntüsü
  alan bir SubAgent. CriticAgent bu görseli değerlendirerek UI kalitesini ölçebilir.

Öncelik 5 — Pre-Warmed Docker Pool
  Her test için sıfırdan container oluşturmak yerine arka planda uykuda bekleyen
  1-2 adet python:3.11-slim container. Executor kodu bu container'a anında atar,
  izole çalıştırır, çıktısını alır. Hem güvenlik hem hız artışı sağlar.
