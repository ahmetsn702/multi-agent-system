1. PROJE NEDİR?
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Bu proje, birden fazla yapay zeka ajanının birlikte çalışarak
yazılım projeleri ürettiği bir orkestrasyon sistemidir.

Kullanıcı bir hedef yazar (örn: "Python ile hesap makinesi yaz"),
sistem bu hedefi 5 farklı ajana böler, her ajan kendi görevini
yapar ve sonunda çalışır durumda bir proje ortaya çıkar.

Gerçek dünya karşılığı: Bir yazılım şirketinde proje yöneticisi,
araştırmacı, yazılımcı, kalite kontrol ve devops ekibinin
otomatize edilmiş hali.


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
2. MİMARİ YAPI
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Kullanıcı Hedefi
      │
      ▼
┌─────────────┐
│  ORCHESTRATOR │ ← Tüm sistemi yöneten beyin
└──────┬──────┘
       │
       ├──▶ PlannerAgent    → Hedefi alt görevlere böler
       ├──▶ ResearcherAgent → İnternette araştırma yapar
       ├──▶ CoderAgent      → Gerçek Python kodu yazar
       ├──▶ CriticAgent     → Kodu puanlar, kalite kontrol
       └──▶ ExecutorAgent   → Kodu çalıştırır, test eder

Her ajan bağımsız çalışır ama MessageBus üzerinden haberleşir.
Orchestrator, görevleri fazlara (phase) böler ve sırayla çalıştırır.

3. 5 AJAN VE GÖREVLERİ

┌────────────────────────────────────────────────────────────┐
│ 1. PLANNER (Planlayıcı)                                    │
│    Dosya: agents/planner_agent.py                          │
│    Model: openai/gpt-oss-120b (OpenRouter)                 │
│    Görev: Kullanıcının hedefini analiz eder, alt görevlere │
│           böler, fazlara ayırır (Phase 1, 2, 3...)         │
│    Çıktı: JSON formatında görev listesi                    │
│    Maliyet: ~$0.001/proje                                  │
├────────────────────────────────────────────────────────────┤
│ 2. RESEARCHER (Araştırmacı)                                │
│    Dosya: agents/researcher_agent.py                       │
│    Model: llama-3.3-70b-versatile (Groq - BEDAVA)          │
│    Görev: İnternette araştırma yapar, kütüphane bilgisi    │
│           toplar, best practice'leri bulur                  │
│    Çıktı: Araştırma raporu (JSON)                          │
│    Maliyet: $0.00 (Groq bedava!)                           │
├────────────────────────────────────────────────────────────┤
│ 3. CODER (Yazılımcı)                                       │
│    Dosya: agents/coder_agent.py                            │
│    Model: openai/gpt-oss-120b (OpenRouter)                 │
│    Görev: Gerçek Python kodu yazar, dosya oluşturur        │
│    Özellik: Mevcut dosyaları kontrol eder (duplikasyon yok)│
│    Çıktı: [FILE:dosya.py]...[/FILE] formatında kod         │
│    Maliyet: ~$0.001/görev                                  │
├────────────────────────────────────────────────────────────┤
│ 4. CRITIC (Eleştirmen)                                     │
│    Dosya: agents/critic_agent.py                           │
│    Model: llama-3.3-70b-versatile (Groq - BEDAVA)          │
│    Görev: Kodu 5 kritere göre puanlar (1-10):              │
│           - Doğruluk (correctness)                         │
│           - Kalite (quality)                               │
│           - Test kapsamı (test_coverage)                   │
│           - Mimari (architecture)                          │
│           - Güvenlik (security)                            │
│    Karar: ≥7.0 = ONAYLA, 4-7 = REVİZYON, <4 = YENİDEN    │
│    Maliyet: $0.00 (Groq bedava!)                           │
├────────────────────────────────────────────────────────────┤
│ 5. EXECUTOR (Yürütücü)                                     │
│    Dosya: agents/executor_agent.py                         │
│    Model: openai/gpt-4o-mini (OpenRouter)                  │
│    Görev: Kodu subprocess ile gerçek çalıştırır,           │
│           pip install yapar, test sonuçlarını raporlar      │
│    Güvenlik: Tehlikeli komutları engeller (rm, del vs.)     │
│    Maliyet: ~$0.0001/görev                                 │
└────────────────────────────────────────────────────────────┘

│                                                      │

└──────────────────────────────────────────────────────┘

5. GÖREV AKIŞI (Bir Projenin Yolculuğu)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Örnek: "Python ile hesap makinesi yaz"

ADIM 1 — Planlama
  Planner hedefi alır → 3 faza böler:
  Faz 1: Veritabanı/temel yapı
  Faz 2: İş mantığı/fonksiyonlar
  Faz 3: Arayüz/test/export

ADIM 2 — Her görev için döngü:
  ┌──────────────────────────────────────┐
  │ Researcher → internette araştır      │
  │      ↓                               │
  │ Coder → araştırmaya göre kod yaz     │
  │      ↓                               │
  │ Critic → kodu puanla                 │
  │      ↓                               │
  │ ≥7.0? → EVET → Executor (çalıştır)  │
  │         HAYIR → Coder'a geri gönder  │
  └──────────────────────────────────────┘

ADIM 3 — Tüm fazlar bitince:
  → project_summary.txt oluşturulur
  → Maliyet raporu gösterilir
  → Dosyalar workspace/projects/ altına kaydedilir
