───────────────────────────────────────────────────────────────────────────────
  1. CHROMADB RAG SİSTEMİ
───────────────────────────────────────────────────────────────────────────────

✅ YENİ DOSYA: core/vector_memory.py (350+ satır)

ÖZELLIKLER:
- sentence-transformers all-MiniLM-L6-v2 modeli (lightweight, fast)
- ChromaDB persistent storage: workspace/chroma_db/
- 3 ana fonksiyon:
  * add_project(slug, summary, code_snippets, metadata)
  * search_similar(query, n=3, min_similarity=0.3)
  * get_context(query, max_length=2000)

ENTEGRASYON:
- memory_agent.py → search_relevant() güncellendi
  * Önce vector search dener
  * Fallback: keyword matching (mevcut sistem)
  * Log: "[Memory] 🔍 Vector search: 3 sonuç bulundu"

- memory_agent.py → save_project() güncellendi
  * JSON'a kaydetme (mevcut)
  * ChromaDB'ye ekleme (yeni)
  * Log: "[Memory] 🔍 Vector DB'ye eklendi: {slug}"

AVANTAJLAR:
- Semantic similarity (anlamsal benzerlik)
- "Flask JWT" → "FastAPI authentication" bulabilir
- Keyword matching'den %40-60 daha iyi

ÖRNEK KULLANIM:
```python
from core.vector_memory import get_vector_memory

vm = get_vector_memory()
vm.add_project("flask-jwt-api", "Flask JWT authentication", ["code1", "code2"])
results = vm.search_similar("FastAPI authentication", n=3)
# [{"slug": "flask-jwt-api", "similarity": 0.85, ...}]
```

───────────────────────────────────────────────────────────────────────────────
  2. PLAYWRIGHT UI TESTER
───────────────────────────────────────────────────────────────────────────────

✅ YENİ DOSYA: agents/ui_tester_agent.py (350+ satır)

ÖZELLIKLER:
- BaseAgent'tan türetildi
- 3 test modu:
  1. Flask app: uvicorn ile başlat → screenshot → durdur
  2. FastAPI app: uvicorn ile başlat → screenshot → durdur
  3. Static HTML: file:// URL ile aç → screenshot

- Screenshot konumu: workspace/projects/{slug}/screenshots/
- Otomatik tetikleme: HTML/CSS/JS veya Flask/FastAPI dosyası varsa

ENTEGRASYON:
- orchestrator.py → _execute_task() güncellendi
  * Coder tamamlandıktan sonra UI tester çalışır
  * Web projesi kontrolü (has_html, has_flask, has_fastapi)
  * Screenshot path Critic'e iletilir
  * Log: "[Orchestrator] 📸 UI test başlatılıyor..."

- critic_agent.py → act() güncellendi
  * Screenshot varsa 6. kriter ekler: ui_quality
  * Görsel tasarım, renk uyumu, layout değerlendirmesi
  * System prompt güncellendi (ui_quality kriteri)
