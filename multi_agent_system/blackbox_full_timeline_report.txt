15. EN SERT TEST - MINI ERP BACKEND'I
======================================================================
Kullanici aksiyonu:
- FastAPI + SQLite mini ERP backend testi calistirildi.
- Bu testte tum agentlar Sonnet 4.6 kullaniyordu.

Logtaki kritik sonuc:
- Planner birden fazla kez cevap verdi.
- Ancak run hemen su hata ile bitti:
  No tasks generated from plan

Detayli analiz:
- Planner cevap dondugu icin model erisimi problemi yoktu.
- Loglarda planner output sürekli 2000 token limitine vurdu.
- Bu, planner cevabinin buyuk ihtimalle kesildigini gosterdi.
- base_agent._parse_json_response parse fail durumunda None yerine {"raw": text} donduruyor.
- planner_agent bunu dict oldugu icin "parse basarili" sanıyor.
- Sonra mode/tasks/phases bos kaldigi icin orchestrator flat mode'da tasks=[] gorup
  "No tasks generated from plan" hatasina dusuyor.

Kritik teknik tespit:
- Sorun Sonnet 4.6'nin baglanamamasi degil.
- Sorun planner parse-fail durumunun yanlis siniflanmasi ve output kesilmesi.

Ileriye donuk oneri:
- _parse_json_response parse fail'de None dondurmeli
- Planner bos tasks/phases planini retry etmeli
- Planner icin output budget buyutulmeli
- Planner think/act icinde iki kez ayni LLM cagrisi yapmamali
