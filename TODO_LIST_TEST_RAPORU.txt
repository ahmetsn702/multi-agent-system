🔍 ANA SORUN: Unterminated String Literal

Bu hata, LLM'in string'leri düzgün kapatmadığı anlamına gelir:
```python
# YANLIŞ:
with TASKS_FILE.open("r", encoding="utf-
    ^
# String tırnak işareti kapatılmamış!

# DOĞRU:
with TASKS_FILE.open("r", encoding="utf-8") as f:
```

🔍 NEDEN AUTO-FIX ÇALIŞMADI?

Auto-fix'imiz şu pattern'leri düzeltiyor:
1. if b == : → if b == 0:
2. sys.path.insert(, ...) → sys.path.insert(0, ...)
3. == : → == 0:

AMA "unterminated string literal" için pattern YOK!

Bu hata:
- Regex ile düzeltilemez (string'in nerede bitmesi gerektiğini bilemeyiz)
- LLM'in düzeltmesi gerekir
- LLM 2 kez denedi ama başarısız oldu
